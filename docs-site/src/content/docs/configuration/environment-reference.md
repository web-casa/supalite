All configuration lives in `.env` at the repo root. Most fields are also editable via the admin Settings page (which writes back to `.env` atomically).

## Secrets

| Variable | Generated by | Notes |
|---|---|---|
| `POSTGRES_PASSWORD` | setup.sh | Shared by all 12 supabase roles. NOT rotatable through the admin wizard. |
| `JWT_SECRET` | setup.sh | Signs ANON_KEY, SERVICE_ROLE_KEY, every user JWT. Rotatable via Secrets page (cascades). |
| `ADMIN_TOKEN` | setup.sh | Bearer token for admin login. Rotatable. |
| `COOKIE_SIGNING_KEY` | setup.sh | HMAC key for admin session cookie. Rotatable. |
| `PG_META_CRYPTO_KEY` | setup.sh | Studio's stored DB credentials encryption. Rotatable. |
| `ANON_KEY` | setup.sh (signed JWT) | Safe for frontend code; bound to `anon` role. Re-minted when JWT_SECRET rotates. |
| `SERVICE_ROLE_KEY` | setup.sh (signed JWT) | Bypasses RLS; never expose to frontend. Re-minted when JWT_SECRET rotates. |

## Networking

| Variable | Default | Purpose |
|---|---|---|
| `SITE_URL` | `http://localhost:3000` | Your **frontend's** public URL. Used in email templates and as the canonical CORS origin. |
| `API_EXTERNAL_URL` | `http://localhost:8000` | This stack's gateway URL. Used for OAuth `redirect_uri` registration. |
| `CORS_ALLOWED_ORIGINS_REGEX` | (empty → falls back to SITE_URL) | Regex alternation of all allowed frontend origins. See [Multi-frontend](/supalite/configuration/multi-frontend/). |
| `GOTRUE_URI_ALLOW_LIST` | (empty) | Comma-separated additional redirect URIs for OAuth. SITE_URL is implicitly allowed. |
| `POSTGRES_BIND_ADDR` | `127.0.0.1` | Host IP for the `:5432` mapping. `0.0.0.0` exposes Postgres externally. |
| `GATEWAY_HTTP_BIND` | (empty → 0.0.0.0) | Host IP for `:8000`. |
| `CADDY_SITE_ADDR` | `:8000` | Caddyfile's site address. Set to a domain to enable auto-HTTPS. |
| `CADDY_HTTPS_BIND` | (empty → 0.0.0.0) | Host IP for `:80` and `:443` (only with HTTPS override file). |

## Auth (GoTrue)

| Variable | Default | Notes |
|---|---|---|
| `JWT_EXP` | `3600` | User JWT lifetime in seconds. |
| `GOTRUE_DISABLE_SIGNUP` | `false` | Set `true` for invite-only mode. |
| `GOTRUE_EXTERNAL_ANONYMOUS_USERS_ENABLED` | `true` | Allow anonymous sign-in. |
| `GOTRUE_MAILER_AUTOCONFIRM` | `true` | Skip email confirmation. Set `false` once SMTP is configured. |

## SMTP

`GOTRUE_SMTP_HOST`, `GOTRUE_SMTP_PORT`, `GOTRUE_SMTP_USER`, `GOTRUE_SMTP_PASS`, `GOTRUE_SMTP_ADMIN_EMAIL`. Test with the **Send Test** button on the admin Settings → SMTP tab.

## OAuth

For each of GitHub / Google / Apple:

- `GOTRUE_EXTERNAL_<PROVIDER>_ENABLED` — `true` / `false`
- `GOTRUE_EXTERNAL_<PROVIDER>_CLIENT_ID`
- `GOTRUE_EXTERNAL_<PROVIDER>_SECRET`
- `GOTRUE_EXTERNAL_<PROVIDER>_REDIRECT_URI` — usually `{API_EXTERNAL_URL}/auth/v1/callback`

Validate credentials with the **Test credentials** button on the Settings → GitHub/Google tab. Apple uses JWT-based secrets and isn't covered by the test endpoint.

## Backups

S3-compatible storage (AWS / Cloudflare R2 / MinIO / Backblaze B2 / …):

| Variable | Notes |
|---|---|
| `BACKUP_S3_ENDPOINT` | Custom endpoint URL (empty for AWS). |
| `BACKUP_S3_BUCKET` | Required. |
| `BACKUP_S3_REGION` | Default `us-east-1`. R2 uses `auto`. |
| `BACKUP_S3_ACCESS_KEY` / `BACKUP_S3_SECRET_KEY` | Required. |
| `BACKUP_S3_PATH_STYLE` | `true` for MinIO/Ceph. |
| `BACKUP_S3_PREFIX` | pg_dump objects go here (default `backup/`). |
| `BACKUP_S3_PGBACKREST_PATH` | pgBackRest objects go here (default `/pgbackrest`). |
| `BACKUP_SCHEDULE_HOURS` | If set, scheduled pg_dump every N hours. |
| `BACKUP_RETENTION_COUNT` | Keep N newest scheduled backups; older auto-deleted. |

## pgBackRest (opt-in physical backups)

| Variable | Default | Notes |
|---|---|---|
| `PGBACKREST_ARCHIVE_MODE` | `off` | Set `on` to enable WAL archiving. Requires Postgres restart. |
| `PGBACKREST_ARCHIVE_COMMAND` | `pgbackrest --stanza=main archive-push %p` | What Postgres calls per WAL segment. |
| `PGBACKREST_WAL_LEVEL` | `replica` | Required for archiving. |
| `PGBACKREST_MAX_WAL_SENDERS` | `3` | Slots for replication / pgbackrest. |

## Compose / images

| Variable | Default | Notes |
|---|---|---|
| `COMPOSE_PROFILES` | `studio` | Comma-separated. Set empty to skip Studio + meta. |
| `ADMIN_IMAGE` | `ghcr.io/web-casa/supalite-admin:latest` | Override when forking to a private registry. |
| `DB_IMAGE` | `ghcr.io/web-casa/supalite-db:latest` | Same. |
| `DB_DATA_VOLUME` | `supalite_db-data` | The Docker named volume holding Postgres data. |
| `DB_DATA_VOLUME_EXTERNAL` | `false` | Set `true` if migrating from a previous project name (reuses an existing external volume). |
| `HOST_PROJECT_DIR` | (auto) | The host path of this repo. setup.sh fills it in. |
| `SETUP_COMPLETE` | `false` | Marked `true` by the setup wizard on first run. |

## Where the values are used

For every variable, `grep -rn "VAR_NAME" docker-compose*.yml volumes/` shows the exact services that consume it. The admin Settings page exposes the most-edited subset behind a UI; advanced users can hand-edit `.env` directly.
